DiscoPanel can be configured via a YAML config file, environment variables, or both. Environment variables take precedence over the config file.

This page covers the panel's own settings. For per-server settings like server.properties, memory, mod loaders and modpacks, use each server's Configuration tab in the UI. The guides on [server software](/guides/server-software/), [modpacks](/guides/modpacks/) and [performance](/guides/performance/) cover those.

## Config file

DiscoPanel looks for `config.yaml` in these locations (in order):

1. Path passed via `-config` flag
2. Current working directory (`./`)
3. `./config/`
4. `/etc/discopanel/`

If no config file is found, defaults are used.

## Environment variables

Every config option can be set via an environment variable with the `DISCOPANEL_` prefix. Nested keys are separated by underscores:

```yaml
server:
  port: "8080"
```

is equivalent to:

```bash
DISCOPANEL_SERVER_PORT="8080"
```

## The sections that matter

Most installs only ever touch a few of these:

| Section | What it controls |
|---|---|
| `server` | The port and address the panel's web UI listens on. |
| `storage` | Where server files, backups, and temp files live on disk. |
| `proxy` | The Minecraft proxy - domain, ports. See [Proxy & Domains](/guides/proxy/). |
| `auth` | Login: local accounts, self-registration, and [OIDC providers](/guides/oidc/keycloak/). |
| `minecraft.global_config` | Defaults applied to every **new** server. Each server's Configuration tab overrides these per server. |
| `docker` | Advanced: the Docker socket, container labels, and `runtime_image` to pull server images from your own registry mirror. |
