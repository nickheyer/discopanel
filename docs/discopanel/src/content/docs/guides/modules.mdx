---
title: Modules
description: Add-ons that run beside your server - Bedrock crossplay, a 3D web map, a status page, and more.
---

Modules are add-ons that run in their own little container next to a Minecraft server. Each server has a **Modules** tab where you can add them, start and stop them, and find their addresses. Modules can auto-start with the server, and DiscoPanel picks their ports for you.

## Geyser - play from phones and consoles

Geyser lets **Bedrock Edition** players (phones, tablets, consoles, Windows 10/11 edition) join your **Java Edition** server. Add the Geyser module to a server and share the Bedrock address shown in the module's details.

For Bedrock players to log in with their real accounts, install the **Floodgate** plugin or mod on the server itself (it's on Modrinth - you can add it via the **Modrinth Projects** field or upload it). Without it, Geyser connects players in offline mode.

:::note
Bedrock uses its own port (UDP). If friends connect from outside your network, that port needs to be forwarded on your router - it doesn't go through the Minecraft proxy.
:::

## BlueMap - a 3D map in your browser

BlueMap renders your world into an interactive 3D map you can open in a web browser and share with your players. It reads the world read-only, so it can't hurt anything. Give it a couple of minutes after the first start - rendering a whole world takes a while.

## Status Panel

A small live dashboard for one server - player count, TPS, memory - that you can share with people who shouldn't have access to DiscoPanel itself.

## Prometheus Exporter

Publishes server stats (status, players, response time) in Prometheus format for people who run monitoring dashboards like Grafana. If you don't know what that means, you don't need it.

## Playit.gg - a free public address

[playit.gg](https://playit.gg) gives your server a free public address that tunnels to your machine, so friends can join **without port forwarding** and without you exposing your home IP. The module runs the official playit agent and feeds tunnel traffic through the DiscoPanel proxy, which means wake-on-connect and autopause keep working.

1. On [playit.gg](https://playit.gg), create an account and generate an **agent secret key**.
2. Add the Playit.gg module to your server, paste the key into the **Agent secret key** field on the module's Configuration tab, and start it. The module won't let you create it without the key.
3. On the playit dashboard, create a **Minecraft Java** tunnel and set its local address to `127.0.0.1:25565` - the agent and the bridge live in the same container.
4. Share the tunnel's public address (something like `yourname.craft.ply.gg`) with your players.

Voice mods that use UDP (Simple Voice Chat, Plasmo Voice) work too: add a UDP tunnel on the playit dashboard pointing at `127.0.0.1:24454`.

## Steam Bridge - friends join through Steam

Steam Bridge exposes your server over **Steam's own network** (the same relay tech Steam games use). Nobody port forwards anything and your IP stays hidden - players connect through Steam using the bridge's SteamID64 instead of an IP address. Two catches: every player needs a small helper on their machine (no mods required, see below), and the module needs its **own throwaway Steam account**.

1. Create a fresh Steam account just for the bridge. Never use your personal account - the bridge runs as Steam AppID 480 ("Spacewar") and stays logged in around the clock.
2. Add the Steam Bridge module to your server and fill in the **Steam username** and **Steam password** fields on its Configuration tab. Both are required unless you flip on the external session toggle.
3. Start the module. When Steam needs a **Steam Guard code**, the module dialog shows an input prompt at the top. Enter the current code from the account's email or authenticator app and login retries automatically. The session is remembered afterwards.
4. The logs and the module's status endpoint show the bridge's **SteamID64**. Share that number with your players.

### Joining as a player

1. Download **DiscoPanel Bridge** for your OS from the [releases page](https://github.com/nickheyer/discomodule-releases/releases) - a single small program, nothing to install and no mods needed. Works with any Minecraft version and any launcher.
2. Make sure Steam is running (your normal account is fine), then run DiscoPanel Bridge and paste the server's SteamID64 when it asks.
3. Keep it open and join `localhost` in Minecraft's multiplayer screen. That's it.

Prefer a mod? The open source [Steam Bridge mod](https://github.com/Ragalikx/steam-bridge-mc) also works with this module for the Minecraft versions it supports.

The **Access policy** field controls who may connect: everyone, friends (friends of the bridge's Steam account - have players friend it), or allowlist (only the SteamID64s you list). Voice mod traffic tunnels through Steam automatically.

Connections flow through the DiscoPanel proxy like any other join, so wake-on-connect, autopause, and proxy stats all keep working. Expect the first start to take a few minutes while the Steam client downloads itself into the module's data folder.

## Modules that ask for input

Some modules need a value from you while they run, like the Steam Guard code above. A module signals this by serving a small prompt endpoint on its health port, and the panel surfaces it as a card at the top of the module dialog. Type the value, hit submit, and the panel hands it back to the module.

Building your own module? Serve `GET /prompt` returning the pending prompt as JSON (or `204` when nothing is waiting), and accept `POST /prompt` with `{"id","value"}`. The Go helper in `pkg/moduleprompt` does this for you. Set `supports_prompts` to `true` in the template metadata so the panel knows to poll.

## Good to know

- Modules get their web ports assigned automatically (from the range in the panel's `module` config, `8100-8199` by default). The exact URL for each module is shown in its details.
- A module belongs to one server. Two servers that both want BlueMap each get their own copy.
- Module files live inside the server's folder under `modules/`, so they're covered by whatever you do with the server folder.
